Theory

The Game of Life (or Life for short) is a cellular automaton created in 1970 by John Horton Conway and is probably the most famous of them so far.
It is a zero-player game, meaning it requires no input and is entirely determined by its initial state and its rules set, which are both trivial.
The game consists of a grid representing Cells, which can only be in one of two states : dead or alive. We'll paint the alive ones in black.

The rules of the original Game are as follows :

Exactly 3 alive neighbours, the Cell is born
(the green cell will be alive the next step)		 Between 2 and 3 alive neighbours, the Cell survives
(the middle cell's state won't change) 		Otherwise, the Cell dies
(the red cell will be dead the next step)

These rules are expressed conventionally as B3/S23 for Birth (3 alive neighbours) / Survive (2 or 3 alive neighbours).
We can change the rules as we please, each set having its own properties, emerging patterns and so forth.

One that I personally like is the B4/S9 rules set applied to 3D, which is really explosive and so requires a tiny population rate (around 0.03).

Controls

  • Grid Width/Height/Depth : Grid dimensions
  • Neighbours : Considered radius in which alive cells are counted
  • Rules Set : Game's rules formatted as "Bx[y[z[...]]]/Sa[b[c[...]]]
  • Max FpS : Maximum Frames per Second
  • Pause/Resume : Pause Game
  • Population Ratio : Percentage of alive cells in the initial state
  • Populate : Reset game
